# 运维手册

## 监控

- 健康检查：`/api/v1/health`
- 推荐指标：
  - 请求量
  - 请求耗时
  - 登录失败次数
  - Agent 超时数
  - 每租户 Agent 并发数

## 日志

- 生产环境输出 JSON 结构化日志
- 已脱敏字段包括密码、密钥、哈希、认证头
- 保留策略建议：
  - 应用日志：30 天
  - 审计日志：至少 180 天
  - 安全事件：若策略要求可保留 365 天

## 备份

- 小规模环境至少每 6 小时一次逻辑备份
- 每日执行一次恢复演练到预发环境
- 生产环境建议启用 WAL 归档或托管 PITR

## 事故处理

- 遇到租户级事故，可直接将 `Tenant.isActive = false`
- 受影响用户执行 refresh token 全量失效
- 如怀疑泄露，轮换 API Key 与 JWT 密钥

## 容灾目标

- 建议 RPO：15 分钟
- 建议 RTO：2 小时
