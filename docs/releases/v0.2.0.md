# Nexus Agent Model Hub v0.2.0

Security hardening, server-side provider config, and frontend component architecture improvements.

## Highlights

- **Auth hardening**: JWT middleware now verifies user existence and active status in the database, not just token validity.
- **HttpOnly cookie sessions**: Frontend auth routes act as a BFF layer; backend tokens are stored in HttpOnly cookies instead of browser localStorage.
- **Encrypted provider config**: Provider API keys are encrypted at rest with AES-256-GCM via `PROVIDER_CONFIG_SECRET`.
- **Provider URL validation**: Base URLs are normalized and validated against known provider endpoints to prevent arbitrary upstream targeting.
- **Configurable rate limiting**: All rate limiter windows and caps are now controlled via environment variables.
- **Frontend component extraction**: Dashboard, chat, and settings pages refactored into focused, reusable components with custom React hooks.
- **Frontend test infrastructure**: Added Vitest configuration and initial test suite for hooks and settings logic.

## Breaking Changes

- `PROVIDER_CONFIG_SECRET` is required in production; omitting it causes provider config operations to fail.
- Frontend no longer stores `accessToken` or `refreshToken` in localStorage; session is maintained via HttpOnly cookies. Existing localStorage sessions will be invalidated on upgrade.
- Provider Base URL must now match the official provider hostname and path. Existing custom Base URLs pointing to non-official endpoints will be rejected.

## New Environment Variables

| Variable                     | Default  | Description                                                   |
| ---------------------------- | -------- | ------------------------------------------------------------- |
| `PROVIDER_CONFIG_SECRET`     | _(none)_ | Encryption key for provider API keys. Required in production. |
| `RATE_LIMIT_WINDOW_MS`       | `900000` | General API rate limit window (ms).                           |
| `RATE_LIMIT_MAX`             | `200`    | Max requests per general rate window.                         |
| `AUTH_RATE_LIMIT_MAX`        | `20`     | Max auth attempts per rate window.                            |
| `AGENT_RATE_LIMIT_WINDOW_MS` | `60000`  | Agent rate limit window (ms).                                 |
| `AGENT_RATE_LIMIT_MAX`       | `30`     | Max Agent calls per rate window.                              |

## Upgrade Notes

- Copy new variables from `.env.example` to your `.env`.
- Set `PROVIDER_CONFIG_SECRET` before deploying to production.
- Re-save any existing provider configurations through `/settings` to encrypt them.
- Clear browser localStorage or log in again to establish the new cookie-based session.
